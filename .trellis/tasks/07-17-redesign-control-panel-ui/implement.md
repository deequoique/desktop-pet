# 执行计划：重构桌宠控制面板 UI 与交互

## 实施顺序

1. 建立可回归的状态与 adapter 边界
   - 为设备选择归一化、多目标结果汇总和通话单目标选择提取纯函数并补测试。
   - 将 `web/src/api.ts` 的普通发送 API 改为显式接收目标 ID 数组并逐设备调用现有单目标协议；通话和 WebRTC 继续只接收单个目标 ID。
   - 验证单设备行为与当前版本一致，再开始替换 UI。

2. 搭建共享外壳与视觉 token
   - 建立左侧四区导航、共享顶部栏、唯一在线状态和 `PeerTargetMenu`。
   - 实现多设备选择记忆、最近上线默认、单设备自动目标和离线禁用态。
   - 在 `styles.css` 建立颜色、间距、圆角、阴影、焦点和响应式 token；暂时保留旧业务区以便对照。

3. 重构控制与发送页
   - 将表情、动作和位置集中到 `ControlView`，实现快捷项与完整动作入口。
   - 将 TTS 与私有音频迁入 `SendView`；保留声音试听、键盘发送、录制、导入、试听、重命名和删除。
   - 接入多目标 command/TTS/audio fan-out，覆盖全成功、部分成功、全失败与目标在发送中离线。
   - 删除跨页面重复入口和重复在线说明。

4. 重构通话页
   - 建立 idle、requesting、calling、in-call、audio-only、error 的分阶段布局。
   - 多台可通话设备时提供单选目标；确保 `call:start`、信令和 hangup 只使用该设备。
   - 保留现有远端屏幕、全屏、麦克风/系统声音控制和 TURN 音频兜底；将轨道与 ICE 细节移出主界面。

5. 收纳设置与诊断
   - 迁移成员改名、服务器/密钥、连接、设备状态/认领、本机缩放、恢复默认、BYOK 和导出日志。
   - 用界面内表单/确认层替换 `prompt`/`confirm`，保持错误和保存反馈可见。
   - 确认废弃浏览器入口没有新增产品分支或维护负担。

6. 清理、响应式与无障碍
   - 删除旧长页面 JSX、失效样式和解释设计的文案。
   - 验证窄窗口、内容溢出、键盘焦点、菜单关闭逻辑、reduced-motion 和通话中导航标记。
   - 对照三张已确认原型做 Electron 实机视觉验收，并记录合理差异。

## 自动验证

```bash
npm test --prefix server
npm run build:web
npm run build:pet
```

若 web 包新增测试脚本，再运行对应组件/纯函数测试；不为了本任务引入重量级测试框架。

## Electron 手工验收矩阵

1. 在线目标
   - 对方 0/1/2+ 台桌宠设备在线。
   - 首次多设备默认最近上线、记住上次选择、新设备不自动加入。
   - 已选设备离线、只剩一台自动切换、全部离线禁用发送。

2. 控制与发送
   - 表情、动作、位置对单设备与多设备投递。
   - TTS 全成功、部分失败、队列满和重试保留输入。
   - 私有音频录制/导入/试听/重命名/删除，以及多设备播放。
   - 首页只出现一处在线状态，控制与发送没有重复功能入口。

3. 通话
   - 单设备直接呼叫；多设备必须单选且只通知目标设备。
   - 接通、结束、拒绝/超时、目标断连、屏幕权限失败、仅音频兜底和恢复超时。
   - 通话中切换页面不结束媒体，返回通话页状态与画面一致。

4. 设置与窗口
   - 双方名称编辑、连接/断开、设备认领、缩放/复位、BYOK 与诊断导出。
   - 常规尺寸、项目允许的最窄窗口、Windows 缩放场景下无横向溢出或不可点击区域。
   - 键盘完成导航、目标选择、二级 tab、发送和通话控制。

## 审查门槛

- 先完成 adapter/选择规则测试，再审查 UI；多目标 fan-out 未验证前不接入 TTS 和音频。
- 通话页完成后单独回归现有 WebRTC 状态机，不以静态视觉验收代替媒体测试。
- 最终检查必须确认 server 事件仍为单目标、接收方无法读取他人音频库、日志不包含密钥/API Key/音频。
- 不修改或暂存用户现有 `docs/releases.md`、`docs/troubleshooting.md` 改动。

## 回滚点

- 步骤 1 可独立回滚到单目标 adapter，不影响 server。
- 步骤 2–5 按页面分段提交；任一页面回归时可恢复旧 JSX，同时保留已验证的共享外壳。
- 若多目标 TTS/音频导致不可接受的队列压力，先在 UI 限制为单选，server 无需回滚。
- 若通话重构影响媒体状态机，恢复旧 `CallView` 呈现，不改动已验证的目标选择与其他页面。
