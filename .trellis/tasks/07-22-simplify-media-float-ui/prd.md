# Simplify floating media window UI

## Goal

让系统级通话浮窗成为不遮挡内容的纯媒体画面窗口，并让控制面板的通话按钮更紧凑。

## Background

- 当前嵌入视图和浮窗复用同一个 `mediaStage`，浮窗通过 portal 渲染，见 `web/src/App.tsx:1522` 与 `web/src/App.tsx:1687`。
- 当前浮窗仍渲染 `.media-controls`，包括屏幕、摄像头、返回、结束等按钮，见 `web/src/App.tsx:1533`。
- Electron 浮窗已有 320×180 最小尺寸且允许自由缩放，见 `pet/src/main/index.js:616`。

## Requirements

- 系统浮窗只显示屏幕共享和摄像头画面，不显示任何按钮、状态卡片、占位宠物或其他操作 UI。
- 控制操作继续保留在主控制面板；关闭系统浮窗仍使用原生窗口关闭按钮，并按现有行为回到控制面板且不结束通话。
- 浮窗保持自由缩放，不锁定宽高比；最小尺寸保持 320×180。屏幕或摄像头画面根据当前浮窗尺寸自动使用全部可用显示区域，并以 `contain` 方式完整显示、不裁切；比例不一致时允许留空边。
- 主控制面板通话画面上的控制按钮整体缩小，并统一按钮高度、字体、内边距和间距；按钮 UI 在不同控制面板尺寸下保持一致的紧凑比例，功能和权限模型不变。
- 屏幕与摄像头的主次画面、隐藏和自动顶替逻辑不变。

## Acceptance Criteria

- [x] 打开系统浮窗后，DOM 中没有 `.media-controls`，可见内容仅包含当前屏幕/摄像头媒体 surface。
- [x] 浮窗可自由改变宽高，不被锁定为 16:9，最小尺寸不低于 320×180；媒体区域始终占用完整浮窗客户区，视频内容完整且不裁切。
- [x] 关闭浮窗后媒体视图回到控制面板，通话不中断。
- [x] 控制面板中的通话按钮明显小于当前版本，且在窄窗口中不遮挡主要媒体画面。
- [x] 屏幕暂停且摄像头可用时仍自动以摄像头为主；屏幕恢复后仍恢复用户偏好的主次布局。
- [x] `npm test --prefix pet`、`npm run build:web` 与 `npm run build:pet` 通过。

## Out of Scope

- 不修改摄像头/屏幕 WebRTC 信令、媒体权限、双方开关权或 TURN audio-only 规则。
- 不增加浮窗内的悬浮、自动隐藏或右键控制菜单。
