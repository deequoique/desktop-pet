# 修复桌宠缩放入口并导出诊断日志

## Goal

提供可靠缩放控制、默认大小恢复和脱敏诊断日志导出，以定位 Win10 异常变大问题。

## Requirements

- 在控制面板提供不依赖透明桌宠命中区域的尺寸调节，并显示当前缩放值。
- 提供“恢复默认大小”，将 scale 恢复为 1，并以可见工作区重新定位桌宠。
- 保留桌宠浮层的现有 +/- 入口，但所有入口必须复用同一套主进程缩放与持久化逻辑。
- 主进程持久记录诊断日志，覆盖启动、应用/OS/Electron 版本、显示器 bounds/workArea/scaleFactor、窗口 bounds、保存 scale、缩放请求与结果、显示器指标变化和未捕获错误。
- 托盘和控制面板提供“导出诊断日志”，由用户选择保存位置。
- 导出内容必须对房间密钥、API Key、凭据、音频内容及其他敏感值脱敏，不包含用户私有音频文件。
- 在取得 Win10 日志前不猜测特定根因；实施可由现有代码证据支持的 DPI/边界修正，并保留诊断信息供后续定位。

## Confirmed Facts

- `pet/src/main/index.js:503-516` 是唯一运行时缩放窗口的主进程入口，收到 scale 后按 360×480 计算 bounds 并保存。
- `pet/src/renderer/main.ts:1195-1238` 的 +/- 控件依赖透明窗口右上角命中和 IPC；Win10 故障机器目前没有可导出的日志。
- 未发现定时或远程自动放大代码；可能原因包括透明窗口命中失败、保存状态异常或 Windows DPI/多显示器 scaleFactor 变化，尚无证据确定。

## Acceptance Criteria

- [x] 控制面板可调节桌宠大小并立即显示实际结果，重启后保持正确 scale。
- [x] “恢复默认大小”将 scale 设为 1，并保证窗口落在当前可见工作区。
- [x] 浮层与控制面板缩放入口不会维护互相冲突的本地 scale。
- [x] 日志记录每次缩放的请求值、钳制值、前后 bounds 和关联显示器 scaleFactor。
- [x] 用户可从托盘或控制面板导出可读诊断包；取消导出不会报错或改变状态。
- [x] 自动化检查确认导出内容不含房间密钥、API Key 或音频数据。
- [x] pet 构建通过，并在可用环境进行窗口缩放手工验证。

## Out of Scope

- 在没有故障机日志或可复现步骤时声称已修复某个未经证实的 Win10 根因。
- 收集或自动上传遥测；诊断日志只在用户主动导出后由用户自行提供。
